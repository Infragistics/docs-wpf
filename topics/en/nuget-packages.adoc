== Infragistics NuGet Packages

As of the 2023.2 release, the Infragistics traditional installer has been removed, and so the WPF controls are now supported exclusively through NuGet moving forward.

Infragistics {ProductName} provides NuGet packages for each control and component in the product. Use the below table to find which packages you need to add to your {PlatformName} application for each control.

There are two ways you can consume the NuGet packages.  You can store the package files in a directory on your machine and link:nuget-feeds.html#adding_local_package[add a package source] pointing to that directory or you can use the licensed link:nuget-feeds.html#adding_infragistics_feed[Infragistics NuGet feed]. This licensed feed requires an Infragistics account linked to a valid product key. Alternatively, a trial version of the NuGet packages are now available on the default public NuGet feed https://api.nuget.org/v3/index.json.

- With the 2019.1 release, the NuGet packages listed below are supported in the .NET Core 3.0 environment.
- As of the 2020.2 release, we no longer support .NET Core 3.0, we support the .NET Core 3.1 and .NET 5.0 .
- As of the 2021.2 release, we support .NET 6.0.
- As of the 2022.2 release, we support .NET 7.0.
- As of the 2023.2 release, we support .NET 8.0.
- As of the 2024.2 release, we support .NET 9.0. We no longer support .NET 3.1, .NET 5, or .NET 7.
- As of the 2025.2 release, we support .NET 10.0. We no longer support .NET 6.0.

We also support .NET Framework 4.X using any of the above versions.

If you want to use the toolbox in Visual Studio with the Infragistics {ProductName} controls, you will need to add the corresponding NuGet packages to the project first, as the toolbox will populate based on the packages that are installed to that project. A full list of which NuGet packages contain which controls can be found below:

=== Misc. Dependencies

[cols="a,a"]
|====
|NuGet Package|Description

| Infragistics.WPF.DataManager
| This package contains shared resources used by many of the LOB controls. E.g. XamDataTree, XamFormulaEditor, etc.

| Infragistics.WPF.DataVisualization
| This package contains shared resources used by all of the data visualization controls. E.g. XamDataChart, XamRadialGauge, etc.

| Infragistics.WPF.Documents.Core
| This package contains many core resources that the Excel Framework requires.

|
Infragistics.WPF.RichTextDocument +
Infragistics.WPF.RichTextDocument.Html +
Infragistics.WPF.RichTextDocument.Rtf +
Infragistics.WPF.RichTextDocument.Word
| The Infragistics.WPF.RichTextDocument package provides the RichTextDocument which is required by the XamRichTextEditor control.

The other packages provide different serializers which can be used to load/save HTML, RTF and Microsoft Word files.

|
Infragistics.WPF.TextDocument +
Infragistics.WPF.TextDocument.CSharp +
Infragistics.WPF.TextDocument.TSql +
Infragistics.WPF.TextDocument.VisualBasic
| These packages are the heart of the link:ig-spe-overview[Syntax Parsing Engine].

Infragistics.WPF.TextDocument is a required package when using the XamSyntaxEditor control. The other packages contain predefined languages that are ready to use right out of the box.

|
Infragistics.WPF.Olap +
Infragistics.WPF.Olap.Adomd +
Infragistics.WPF.Olap.Excel +
Infragistics.WPF.Olap.FlatData +
Infragistics.WPF.Olap.Xmla +
Infragistics.WPF.Olap.Xmla.Oracle +
Infragistics.WPF.Olap.Xmla.Sap

| The Infragistics.WPF.Olap package contains shared resources used by the other Olap packages as well as the XamPivotGrid control.

The Infragistics.WPF.Olap.Adomd package provides functionality for connecting to an ADOMD.NET data source.

The Infragistics.WPF.Olap.Excel package provides functionality for connecting to an Excel spreadsheet.

The Infragistics.WPF.Olap.FlatData package provides functionaliy for connecting to a flat list of data.

The Infragistics.WPF.Olap.Xmla package provides functionality for connecting to an Xmla data source.

The Infragistics.WPF.Olap.Xmla.Oracle package provides functionality for connecting to an Oracle Xmla data source.

The Infragistics.WPF.Olap.Xmla.Sap package provides functionality for connecting to an Sap Xmla data source.
