= What's New in 2022 Volume 1

=== Data Legend

The link:{DataChartLink}.{DataLegendName}.html[{DataLegendName}] is a new component that works much like the `Legend`, but it shows values of series and provides many configuration properties for filtering series rows and values columns, styling and formatting values. This legend updates when moving the mouse inside of the plot areas the various Infragistics chart controls.

The full documentation topics for the link:{DataChartLink}.{DataLegendName}.html[{DataLegendName}] can be found at the following links:

- link:categorychart-data-legend.html[{CategoryChartName} Data Legend]
- link:datachart-data-legend.html[{DataChartName} Data Legend]
- link:financial-chart-data-legend.html[{FinancialChartName} Data Legend]

=== Data Tooltip Layer

The link:{DataChartLink}.DataToolTipLayer.html[DataToolTipLayer] is a new annotation layer that displays values and titles of series as well as legend badges of series in a tooltip. In addition, it provides many configuration properties of the link:{DataChartLink}.{DataLegendName}.html[{DataLegendName}] for filtering series rows and values columns, styling, and formatting values. The tooltip's value updates while moving the mouse inside of the plot area of the various Infragistics chart controls.

The link:{DataChartLink}.DataTooltipLayer.html[DataToolTipLayer] is now the default tooltip type for each of the Infragistics chart controls. Each of the properties that exist on the link:{DataChartLink}.DataTooltipLayer.html[DataToolTipLayer] in the link:{DataChartLink}.{DataChartName}.html[{DataChartName}] is available on the link:{DataChartLink}.{CategoryChartName}.html[{CategoryChartName}], link:{FinancialChartLink}.{FinancialChartName}[{FinancialChartName}] and link:{DataChartLink}.{ShapeChartName}.html[{ShapeChartName}] controls as well, and is prefixed by `DataTooltip`.

The full documentation topics for the link:{DataChartLink}.DataTooltipLayer.html[DataToolTipLayer] can be found at the following links:

- link:categorychart-data-tooltip.html[{CategoryChartName} Data Tooltip]
- link:datachart-data-tooltip.html[{DataChartName} Data Tooltip]
- link:financial-chart-data-tooltip.html[{FinancialChartName} Data Tooltip]

=== Data Chart:

 - Added animation and transition-in support for Stacked Series. Animations can be enabled by setting the link:{DataChartLink}.categoryseries{ApiProp}IsTransitionInEnabled.html[IsTransitionInEnabled] property to true. From there, you can set the link:{ApiPlatform}{CategoryChartAssembly}{ApiProp}{CategoryChartNamespace}.xamdomainchart{ApiProp}TransitionInDuration.html[TransitionInDuration] property to determine how long your animation should take to complete and the link:{DataChartLink}.categoryseries{ApiProp}TransitionInMode.html[TransitionInMode] to determine the type of animation that takes place.

=== Category Chart, Financial Chart, DataChart 

 - Added link:{DataChartLink}.categoryseries{ApiProp}assigningcategorystyle_ev.html[AssigningCategoryStyle] event, is now available to all series in DataChart. This event is handled when you want to conditionally configure aspects of the series items such as `Fill` background-color and highlighting.
 - New link:{DataChartLink}.calloutlayer{ApiProp}AllowedPositions.html[AllowedPositions] enumeration for CalloutLayer. Used to limit where the callouts are to be placed within the chart. By default, the callouts are intelligently placed in the best place but this used to force for example TopLeft, TopRight, BottomLeft or BottomRight.
 - New corner radius properties added for Annotation Layers; used to round-out the corners of each of the callouts. Note, a corner radius has now been added by default.
	* link:{DataChartLink}.calloutlayer{ApiProp}CalloutCornerRadius.html[CalloutCornerRadius] for CalloutLayer:	
	* link:{DataChartLink}.finalvaluelayer{ApiProp}AxisAnnotationBackgroundCornerRadius.html[AxisAnnotationBackgroundCornerRadius] for FinalValueLayer
	* link:{DataChartLink}.crosshairlayer{ApiProp}XAxisAnnotationBackgroundCornerRadius.html[XAxisAnnotationBackgroundCornerRadius] and link:{DataChartLink}.crosshairlayer{ApiProp}YAxisAnnotationBackgroundCornerRadius.html[YAxisAnnotationBackgroundCornerRadius] for CrosshairLayer
 - New Added link:{ApiPlatform}{CategoryChartAssembly}{ApiProp}{CategoryChartNamespace}.xamdomainchart{ApiProp}HorizontalViewScrollbarMode.html[HorizontalViewScrollbarMode] and link:{ApiPlatform}{CategoryChartAssembly}{ApiProp}{CategoryChartNamespace}.xamdomainchart{ApiProp}VerticalViewScrollbarMode.html[VerticalViewScrollbarMode] enumeration to enable scrollbars in various ways. When paired with link:{ApiPlatform}{CategoryChartAssembly}{ApiProp}{CategoryChartNamespace}.xamdomainchart{ApiProp}IsVerticalZoomEnabled.html[IsVerticalZoomEnabled] or link:{ApiPlatform}{CategoryChartAssembly}{ApiProp}{CategoryChartNamespace}.xamdomainchart{ApiProp}IsHorizontalZoomEnabled.html[IsHorizontalZoomEnabled], you'll be able to persist or fade-in and out the scrollbars along the axes to navigate the chart.
 - New link:{DataChartLink}.numericaxisbase{ApiProp}FavorLabellingScaleEnd.html[FavorLabellingScaleEnd], determines whether the axis should favor emitting a label at the end of the scale. Only compatible with Numeric axes (e.g. NumericX, NumericY, PercentChangeY).
 - New * link:{DataChartLink}.splineseries.html[IsSplineShapePartOfRange] property on SplineSeries determines whether to include the spline shape in the axis range requested of the axis.
 - New link:{CategoryChartLink}.{CategoryChartName}{ApiProp}XAxisMaximumGap.html[XAxisMaximumGap], determines the maximum allowed value for the plotted series when using XAxisGap. The gap determines the amount of space between columns or bars of plotted series.
 - New link:{CategoryChartLink}.{CategoryChartName}{ApiProp}XAxisMinimumGapSize.html[XAxisMinimumGapSize], determines the minimum allowed pixel-based value for the plotted series when using XAxisGap to ensure there is always some spacing between each category.
