== 不明な値の補間 - 概念的概要

=== 概要

不明な値の補間機能により、 _xamSparkline_   は不明な値 (`Null` または `double`.`NaN``)` を検出し、指定された補間アルゴリズムを通して不明な値のスペースをレンダリングできます。データに不明な値があり、この機能を使用しない場合は、補間が指定されていないと見なされ、不明な値はプロットされません。

データ セットに不明な値がある場合は、不明な値の補間機能で以下のオプションを指定できます。

* __不明な値を直線的に補間するには_  、_ null 値ポイント前のデータ ポイントを null 値ポイント後のデータ ポイントに接続します。

* __不明な値をプロットせずに_  、_ データ コレクションに存在する null 値のブランクのポイントが挿入されます。

.注:
[NOTE]
====
線形補間オプション (上記の最初のオプション) がデータ ポイントに対して機能するには、前と次のデータ ポイントの両方が存在していなければなりません。たとえば、null 値がコレクションの最初の値の場合、接続する前のポイントがないことになり、その結果、null 値がブランクのポイントとして表示されます。最後のデータ ポイントが null の場合も同様のことが当てはまります。
====

不明な値の補間機能は、 link:{ApiPlatform}controls.charts.xamsparkline{ApiVersion}~infragistics.controls.charts.xamsparkline~unknownvalueplotting.html[UnknownValuePlotting] プロパティにより有効になり、(補間のタイプという意味で) 設定されます (<<_Ref317177339,プロパティ設定>>のブロックを参照)。

[[_Ref317177334]]

=== データ要件

`Null` 値または `double.NaN` 値のみ不明と見なされます。期待される値のタイプは、数値 (整数または double) です。数値以外の値は、不明とは見なされません。

[[_Ref317177339]]

== 不明な値の補間 - コード例

[[_Ref317177387]]

=== 説明

以下のコードは、線形補間オプションを使用した不明な値の処理を示しています。

[[_Ref317177391]]

=== コード

*XAML の場合:*

[source,xaml]
----
<ig:XamSparkline Name="xamSparkline1" UnknownValuePlotting="LinearInterpolate"/>
----

*C# の場合:*

[source,csharp]
----
this.xamSparkline1.UnknownValuePlotting = UnknownValuePlotting.LinearInterpolate;
----

*Visual Basic の場合:*

[source,vb]
----
Me.xamSparkline1.UnknownValuePlotting = UnknownValuePlotting.LinearInterpolate
----
