== UndoUnits を元に戻す履歴に追加するためのメソッド参照

[[_Ref320837974]]

=== 概要

UndoManager クラスは、UndoUnit を構成し、作業単位を元に戻す履歴に追加する多数のヘルパー メソッドを提供します。

[[_Ref320837982]]

=== メソッド参照の概要

以下の表で、UndoUnits を元に戻す履歴に追加するための UndoManager メソッドの目的と機能を簡単に説明します。

[options="header", cols="a,a"]
|====
|メソッド|説明

| link:{ApiPlatform}undo{ApiVersion}~infragistics.undo.undomanager~addchange.html[AddChange]
|これは、 link:{ApiPlatform}undo{ApiVersion}~infragistics.undo.undounit_members.html[UndoUnit] 派生クラス オブジェクトを元に戻す履歴に追加するために使用される link:{ApiPlatform}undo{ApiVersion}~infragistics.undo.undomanager_members.html[UndoManager] メソッドです。 

このメソッドには、多数のオーバーロードがあります。 

AddChange(UndoUnit) を除くすべてのオーバーロードでは、変更を表す UndoUnit は履歴にそれ自体を自動的に追加します (それ自体を渡す `AddChange` を呼び出す)。したがって、これらのデリゲートによって同じ `AddChange` メソッドが起動されることにはならないはずです。 

これらのメソッドは、UndoUnit インスタンスの作成を、履歴が更新される UndoManager の link:{ApiPlatform}undo{ApiVersion}~infragistics.undo.undounitfactory_members.html[UndoUnitFactory] に最終的にデリゲートします。呼び出しをこれらのメソッドに変更することなく、これらの操作に使用される UndoUnit インスタンスを変更することができます。

| link:{ApiPlatform}undo{ApiVersion}~infragistics.undo.undomanager~addcollectionchange.html[AddCollectionChange]
|このメソッドは、履歴のコレクション変更のために UndoUnit を追加します。 

タイプ _ICollection_ <T> のコレクションと、そのコレクション内で発生した変更のタイプに関する情報をパラメーターとして取得するオーバーロードがいくつかあります。

| link:{ApiPlatform}undo{ApiVersion}~infragistics.undo.undomanager~addpropertychange.html[AddPropertyChange]
|このメソッドは、元に戻す履歴への指定されたプロパティ値変更のために link:{ApiPlatform}undo{ApiVersion}~infragistics.undo.propertychangeundounitbase_members.html[PropertyChangeUndoUnitBase] を追加します。 

このメソッドにはパラメーターとして取得する多数のオーバーロードがあります: 

* プロパティが変更されたオブジェクトへの参照 

* プロパティの古い値。 

* プロパティの新しい値。 

* 変更されたプロパティを識別する追加情報
