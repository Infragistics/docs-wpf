== プロジェクト計算

=== プロジェクト計算

Microsoft Project と同じように、 _xamGantt_   コントロールは各編集の後でプロジェクトを計算し、プロジェクトおよびタスク データを最新状態に保つための機能を提供します。

_xamGantt_   コントロールの組み込みのスケジューリング エンジンは、Microsoft Project と同じスケジューリング機能を多数提供します。

このエンジンは以下を実行します。

* 内部タスク更新 - タスク プロパティを変更する時にタスク内の関連プロパティを更新します。
* タスク間更新 - 関連する先行処理/後続処理だけでなく先祖と子孫を更新します。

デフォルトで、スケジューリング エンジンは各編集の後でプロジェクトの再計算を実行しますが、これは構成可能な機能で、プロジェクト計算を明示的にトリガーするように設定できます。

== プロジェクト クリティカル パス計算

=== プロジェクト クリティカル パス計算

Microsoft Project と同じように、 _xamGantt_   コントロールはプロジェクト クリティカル パス計算のための機能を提供します。

プロジェクト クリティカル パスは、計算されたプロジェクトの開始日または終了日にとってクリティカルなタスクで構成されます。定義によって、遅延した結果、期限または制約を違反するまたはプロジェクトの期間を増やすことになる場合、タスクはクリティカルです。

_xamGantt_   コントロールの使用:

* 単一または複数のクリティカル パスを強調表示できます
* タスクのクリティカル スラック制限を構成します

==== 関連トピック:

* link:xamgantt-configuring-critical-slack-limit.html[クリティカル スラック制限を構成]

== プロジェクト開始日または終了日からスケジュール

=== プロジェクト開始日または終了日からスケジュール

Microsoft Project と同じように、 _xamGantt_   コントロールはプロジェクトの開始日または終了日からプロジェクトをスケジュールするための機能を提供します。

これはプロジェクト タスクの開始日または終了日への更新はプロジェクトの開始日または終了日に基づいて発生することを意味します。

* 開始日からスケジュールされたプロジェクトでは、スケジューリング エンジンはプロジェクトの終了日を計算するために開始ポイントとして開始日を使用します。
* 終了日からスケジュールされたプロジェクトでは、スケジューリング エンジンはプロジェクトの開始日を計算するために開始ポイントとして終了日を使用します。
