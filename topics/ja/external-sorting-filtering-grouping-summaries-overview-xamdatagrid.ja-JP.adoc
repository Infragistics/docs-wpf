== はじめに

=== xamDataGrid の外部ソート、フィルタリング、グループ化、および集計

XamDataGrid は、外部での並べ替え、フィルタリング、グループ化、集計機能を含み、操作を外部ソースへオフロードできます。デフォルトでは、データ プレゼンターによって並べ替え、フィルタリング、グループ化、集計計算が内部で実行されます。内部処理でデータ プレゼンターがこれらの処理を独自のオブジェクトモデルおよび UI と統合することが許可されている場合でも、これらの処理を外部で実行する利点もあります。

[options="header", cols="a,a"]
|====
|ケース|説明

|メモリ割り当ての時間へのパフォーマンスの影響を防止します。
|並べ替え、フィルタリング、グループ化、集計計算の内部処理では、データ プレゼンターが `DataRecord` オブジェクトを演算の一部であるデータ項目のすべてに割り当てをします。 

これは、データ レコード オブジェクトの割り当て処理によってメモリ割り当ての時間と同様に初期 CPU 時間でパフォーマンスの影響があります。 

外部処理の利点は、パフォーマンスがより向上することです。

|ネイティブ XamDataGrid UI で外部データ ソースの使用
|外部プロセスでデータ ソースは ICollectionView を継承します。ビューで実行される並べ替え、フィルタリング、グループ化操作は、データ プレゼンターの UI に反映されます。 

データ プレゼンターを介した並べ替えまたはグループ化条件への変更は、基になる link:http://msdn.microsoft.com/ja-jp/library/ms644316[ICollectionView] にも反映されます。

|`ICollectionView` インスタンスを他の View コントロールへバインド
|同じ `ICollectionView` インスタンスを他のビュー コントロールへバインドします。これは、データ プレゼンター UI を介して適用される並べ替え、フィルタリング、グループ化条件に反映されます。

|データ プレゼンターは、 link:http://msdn.microsoft.com/ja-jp/library/system.componentmodel.icollectionview.sortdescriptions[SortDescriptions] および (または) link:http://msdn.microsoft.com/ja-jp/library/system.componentmodel.icollectionview.groupdescriptions[GroupDescriptions] が外部で変更されたときに link:{ApiPlatform}datapresenter{ApiVersion}~infragistics.windows.datapresenter.fieldlayout~sortedfields.html[SortedFields] を同期します。
|`ICollectionView` の `SortDescription` および `GroupDescriptions` が外部で変更されると、データ プレゼンターに変更が通知され、`SortedFields` コレクションに変更を反映するために同期されます。 

注: 

[NOTE] 

==== 

ルート フィールドのレイアウトのみが外部処理をサポートします。子フィールドレイアウトに複数のコレクションビューがあるため、子フィールド レイアウトはサポートされません。 

====

|操作をリモートで実行
|サーバー上での並べ替え、フィルタリング、グループ化、集計計算を実行します。データ プレゼンターの機能は、結果のみを表示し、その UI は、サーバーでの並べ替え、フィルタリング、グループ化、および集計のコメントの生成に使用されます。

|====
