== はじめに

=== ルーラー マージンの概要

_xamSyntaxEditor™_   コントロールは、編集領域の幅に沿ってカレットの位置を識別するため赤点でルーラー マージンをサポートします。有効にすると、分割の有効無効に関係なく編集領域の一番上にルーラーが表示されます。ただし link:xamsyntaxeditor-splitting.html[splitting] が有効になっていると、それぞれの分割に対し 1 つで 2 つのルーラーが描画されます。

以下のスクリーンショットは  _xamSyntaxEditor_   で、ルーラー マージンが有効であり、カレットは列 15 (1 ベース) に配置されています。

image::images/xamSyntaxEditor_RulerMargin.png[]

注:

[NOTE]
====
現在の 1 ベースのカレット列は、カレットの読み取り専用 link:{ApiPlatform}controls.editors.xamsyntaxeditor{ApiVersion}~infragistics.controls.editors.caret~characterdisplaycolumn.html[CharacterDisplayColumn] プロパティを使用して取得できます。
====

== 表示／非表示

=== 概要

ルーラー マージンを表示/非表示にします。

=== プロパティ設定

[options="header", cols="a,a,a"]
|====
|目的:|使用するプロパティ:|設定値

|ルーラー マージンを表示します。
|`IsRulerMarginVisible`
|`true`

|ルーラー マージンを非表示にします。
|`IsRulerMarginVisible`
|`false`

|====

注:

[NOTE]
====
link:{ApiPlatform}controls.editors.xamsyntaxeditor{ApiVersion}~infragistics.controls.editors.syntaxeditorcommandtype.html[ToggleRulerMarginVisibility] コマンドを起動することによりルーラー マージンを表示または非表示にすることもできます。
====

=== コード

以下のコードスニペットは、ルーラー マージンを表示にする方法を示しています。

*C# の場合:*

[source,csharp]
----
this.xamSyntaxEditor1.IsRulerMarginVisible = true;
----

*Visual Basic の場合:*

[source,vb]
----
Me.xamSyntaxEditor1.IsRulerMarginVisible = True
----

== 背景ブラシを変更

=== 概要

ルーラー マージン領域を塗りつぶすために使用するブラシを変更します。

=== プロパティ設定

[options="header", cols="a,a,a"]
|====
|目的:|使用するプロパティ:|設定値

|背景ブラシを変更
|`RulerMarginBackground`
| link:http://msdn.microsoft.com/ja-jp/library/system.windows.media.brush.aspx[Brush] タイプのオブジェクト。

|====

=== コード

以下のコード スニペットは、ルーラー マージンの背景を明るい灰色にする方法を示しています。

*C# の場合:*

[source,csharp]
----
this.xamSyntaxEditor1.RulerMarginBackground = new SolidColorBrush(Colors.LightGray);
----

*Visual Basic の場合:*

[source,vb]
----
Me.xamSyntaxEditor1.RulerMarginBackground = New SolidColorBrush(Colors.LightGray)
----
