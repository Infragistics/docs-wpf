= WPF 固有

ほとんどの {ProductName} コントロールはクロスプラットフォームです。つまり、WPF 版と Silverlight 版の各コントロールのオブジェクト モデルと API は互換性があります。コントロールを作成および操作するためのコード (C#/VB.NET コード ビハインドまたは XAML) は、WPF 版のアプリケーションと Silverlight 版のアプリケーションで共有できます。プロジェクトの参照をプラットフォーム固有のバージョンのコントロール アセンブリを参照するように変更するだけです。

ただし、{ProductName} 専用のコントロールがいくつかあります。以下にこれらのコントロールの一覧を示します。

* link:wpf-the-data-presenter-family.html[データ プレゼンター ファミリ]
* link:editors.html[エディター]
* link:wpfspecific-frameworks.html[フレームワーク]
* link:wpfspecific-grids.html[グリッド]
* link:wpfspecific-interactions.html[操作]
* link:wpfspecific-layouts.html[レイアウト]
* link:wpfspecific-menus.html[メニュー]
